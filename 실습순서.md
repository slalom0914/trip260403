#### 코드 설명
주의사항 : 반드시 실행하기 전에 아래 명령을 실행해서 Redis 컨테이너를 실행해야 합니다.
만일 윈도우에 WSL2를 사용하고 있고 그쪽에 redis를 설정하였다면 반드시 host포트번호를
6379가 아니라 6380으로 설정해야 합니다. 
만일 도커 이미지에서 Redis를 띄울 때 포트 설정을 빼먹었다면 반드시 아래처럼 port설정을 명시적으로 할 것. 
그렇지 않으면 Redis에 정보 확인 불가하므로 꼭 주의하세요.

4. refresh token 관련 키만 조회

```bash
# 실행 중인 Redis 컨테이너 d14d19eb1e9c 내부의 Redis 서버에 명령을 내리는 것. 
# 해당 컨테이너 내부의 Redis 서버에 명령을 내리는 것. 
# redis-cli는 Redis명령어를 실행하는 CLI 도구
# scan 0은 Redis key를 순회 검색하는 명령. 0은 처음 검색 시작 커서(cursor)값
# match 'refresh:*' - refresh:로 시작하는 키만 찾겠다는 의미. 예) refresh:kiwi@hot.com
# count 100은 한 번 검색할 때 Redis에게 대략 100개 정도씩 찾아달라는 힌트.- 정확히 100개를 보장하는 것은 아님.
# 이 명령어의 목적은 로그인 코드에서  redisTokenService.saveRefreshToken(email, refreshToken, ttl);식으로 
# 저장된 refresh token을 조회하는 것. 즉 현재 Redis에 저장된 key 중에서 refresh: 로 시작하는 것들을 찾아줘. 
docker exec -it interesting_goldwasser redis-cli scan 0 match 'refresh:*' count 100

docker exec -it interesting_goldwasser redis-cli keys 'refresh:*'
```
4-1. 결과가 어떻게 나오는가?

```bash
1) "0"
2) 1) "refresh:kiwi@hot.com"
   2) "refresh:test@naver.com"
# 결과를 해석하면 첫번째 값 : "0" 은 다음 검색할 커서 값임. "0"이면 검색이 끝났다는 뜻 
# 두 번째 값 : key 목록. refresh: 패턴에 맞는 key들이 나열됨. 
# 즉 위 결과는 두 개의 Refresh Token key가 Redis에 저장되어 있다는 의미임.  
```

4-2. 커서(cursor) 개념 이해 
- SCAN은 한 번에 전체를 다 읽는 방식이 아니라 조금씩 나눠서 읽는 방식 입니다. 
- 예를 들어 첫 실행 결과가 이렇게 나올 수도 있습니다. 

```bash
1) "52"
2) 1) "refresh:test@naver.com"
   2) "refresh:kiwi@hot.com"
# 결과를 해석하면 첫번째 값 : "52" 은 다음 검색할 커서 값임. 즉 아직 검색이 안 끝남 
# 다음에는 커서 52로 다시 검색해야 함. 
# 즉 다음 명령을 이어서 실행 합니다. 
docker exec -it interesting_goldwasser redis-cli scan 52 match 'refresh:*' count 100
# 그리고 최종적으로 첫 번째 값이 "0"이 나오면 끝 입니다. 
```

4-3. 왜 KEYS 대신 SCAN을 쓰는가?

```bash
docker exec -it interesting_goldwasser redis-cli keys 'refresh:*'
# KEYS 명령은 한 번에 전체 key를 다 뒤짐.
# 데이터가 많으면 Redis성능에 부담.
# 운영 환경에서는 권장하지 않음. 
# SCAN은 조금씩 나눠서 검색하므로 서버에 부담이 덜함. 
# SCAN은 운영 환경에서도 상대적으로 안전함. 
```

4-4. docker exec -it d14d19eb redis-cli scan 0 match 'refresh:*' count 100명령으로 알수 있는 것. 
- 알 수 있는 것. 
  - refresh: 패턴의 key가 존재하는지 
  - 어떤 이메일 기준 key가 저장됐는지
  - refresh key 개수가 대략 몇 개 인지
- 알 수 없는 것.
  - 실제 저장된 Refresh Token 값 
  - TTL(남은 만료 시간)

4-5. 다음으로 같이 확인해야 할 명령
예를 들어 scan결과에서 아래 key를 찾았다고 가정하겠습니다. 

```bash
refresh:kiwi@hot.com
```
그 다음 실제 값 확인 

```bash
docker exec -it interesting_goldwasser redis-cli get refresh:kiwi@hot.com
```
TTL 확인
```bash
docker exec -it interesting_goldwasser redis-cli ttl refresh:kiwi@hot.com
```

9. Refresh Token저장 + TTL 설정

```bash

# TTL: 3600초 (1시간)
# 7일(604800초) 동안 Refresh Token 저장

```
